# App Store Assets Examples

## Example 1: Google Play feature graphic

### Input

```text
Use the App Store Assets Design Skill.

Asset type: Google Play feature graphic
Required size: 1024 × 500 px
App name: Crew
Main message: Hop in!
Brand colors: soft green, warm yellow, clean white
Target audience: young European friends planning activities
Style reference: friendly, realistic, not too AI-looking
```

### Expected output pattern

```text
Asset goal:
Communicate that Crew helps friends quickly join and organize activities.

Canvas and safe-area notes:
1024 × 500 px. Keep main text and logo away from edges. Avoid tiny UI text.

Layout direction:
Left: bold headline "Hop in!".
Right: angled phone mockup showing simplified chat/activity UI.
Background: soft abstract shapes using brand colors.

Image generation prompt:
...

Production checklist:
- Export PNG/JPEG under platform limit.
- Check text readability at small preview size.
- Remove detailed addresses from screenshots.
```

## Bad output

```text
Make it colorful and modern with people having fun.
```

Why bad:

- No canvas guidance.
- No safe area.
- No concrete layout.
- No production checklist.

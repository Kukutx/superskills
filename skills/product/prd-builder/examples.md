# PRD Builder Examples

## Example

### Input

```text
Idea: friend activity cost split feature
Target users: young friends planning BBQs and parties
Constraints: simple v1
```

### Expected output pattern

```text
MVP scope:
- Create activity expense
- Add participants
- Show who paid and who owes

Out of scope:
- Automatic bank transfer
- Multi-currency optimization
- Debt collection

Acceptance criteria:
- User can add an expense with amount, payer, participants.
- Balance updates immediately.
```

## Bad output

```text
Build a complete social finance platform.
```

Why bad: too broad for MVP.
